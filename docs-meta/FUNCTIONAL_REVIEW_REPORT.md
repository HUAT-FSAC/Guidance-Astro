## 二、功能缺陷与问题

### 2.1 🔴 高严重度问题

#### 2.1.1 Content Security Policy 使用 'unsafe-inline'

| 项目         | 详情                                                           |
| ------------ | -------------------------------------------------------------- |
| **位置**     | `src/config/security.ts` Line 16-17                            |
| **问题描述** | CSP 配置中使用了 `'unsafe-inline'` 允许内联脚本，存在 XSS 风险 |
| **影响范围** | 全站安全                                                       |
| **复现步骤** | 检查响应头中的 `Content-Security-Policy`                       |
| **修复建议** | 迁移到 nonce-based CSP，需要 Astro middleware 支持             |
| **参考**     | 代码中已标记 `TODO: 迁移到 nonce-based CSP`                    |

#### 2.1.2 环境变量缺少类型定义

| 项目         | 详情                                                                               |
| ------------ | ---------------------------------------------------------------------------------- |
| **位置**     | `src/env.d.ts`                                                                     |
| **问题描述** | Cloudflare 环境变量（DB, SESSION_KV, GITHUB_CLIENT_ID 等）缺少 TypeScript 类型定义 |
| **影响范围** | 开发体验，潜在的类型错误                                                           |
| **修复建议** | 扩展 `Env` 接口定义所有环境变量                                                    |

### 2.2 🟡 中严重度问题

#### 2.2.1 注册页面前端验证与后端不一致

| 项目         | 详情                                                                                                                                                                                           |
| ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **位置**     | `src/pages/register.astro`                                                                                                                                                                     |
| **问题描述** | 1. 前端验证规则：`/^[a-zA-Z0-9_]{3,20}$/`<br>2. 后端验证规则：`/^[a-zA-Z0-9_\u4e00-\u9fff]+$/` 且长度 2-32<br>3. 前端不支持中文用户名，后端支持<br>4. 前端密码验证仅检查长度，后端无此限制说明 |
| **影响范围** | 用户体验，可能导致注册失败                                                                                                                                                                     |
| **修复建议** | 统一前后端验证规则                                                                                                                                                                             |

#### 2.2.2 API 端点缺少限流保护

| 项目         | 详情                                                    |
| ------------ | ------------------------------------------------------- |
| **位置**     | `src/pages/api/**/*.ts`                                 |

| **影响范围** | 安全性                                                  |
| **修复建议** | 添加基于 IP/用户账号的请求限流中间件                    |
