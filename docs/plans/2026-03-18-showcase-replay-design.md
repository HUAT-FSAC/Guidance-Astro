# Showcase Replay Enhancement Design

## 背景

现有 `ShowcaseLab` 已经能完成静态场景切换，但展示方式仍偏“切页式查看”。下一轮增强的目标，是把它升级成一个可讲解、可回放的纯前端演示模块，让首页既能自助浏览，也能在演示场景中自动播放核心流程。

## 目标

1. 为每个展示场景加入时间轴回放。
2. 支持播放、暂停、上一帧、下一帧和进度拖动。
3. 在回放过程中同步更新指标、赛道车辆位置、进度路径、阶段状态和讲解文案。
4. 保持纯前端实现，不依赖后端或实时数据接口。
5. 用户手动操作后暂停自动播放，避免交互冲突。

## 非目标

1. 不做真实物理仿真。
2. 不开放任意参数编辑面板。
3. 不引入外部状态库或图表库。

## 方案选择

### 方案 A：只做自动轮播

仅定时切换场景，成本最低，但无法体现单场景内部的过程变化。

### 方案 B：时间轴回放 + 自动讲解

每个场景包含若干离散帧，播放时按帧推进，并在场景末尾自动进入下一个场景。这一方案能清楚解释“系统如何从起点走到当前状态”，同时仍然保持静态数据驱动。

### 方案 C：参数沙盒

允许用户修改速度、延迟、风险值等参数并实时重算页面。扩展性强，但复杂度和测试成本明显上升。

## 推荐方案

选择方案 B。

## 交互设计

模块新增一块 `Mission Replay` 控制区，包含：

1. 当前帧标题与讲解说明
2. 播放 / 暂停按钮
3. 上一帧 / 下一帧按钮
4. 进度条拖动控件
5. 当前帧序号与播放状态提示

行为规则：

1. 切换场景时，回放重置到该场景第 1 帧。
2. 点击播放后，按固定节奏推进帧。
3. 播放到当前场景末帧后，自动跳转到下一个场景的第 1 帧继续。
4. 用户点击按钮、拖动进度条或切换子系统时，自动播放暂停。

## 数据设计

为每个 `ShowcaseScenario` 新增 `replay`：

1. `frames`：离散帧数组
2. 每帧包含：
    - `id`
    - `title`
    - `summary`
    - `progressPath`
    - `car`
    - `trendIndex`
    - `metricOverrides`
    - `stageOverrides`
3. `frameDurationMs`：自动播放时单帧停留时间

这样可以把“场景静态基线”与“回放动态变化”分离，避免把所有表现直接写死在组件里。

## 实现设计

1. `src/data/showcase-lab.ts`
    - 扩展回放帧类型与每个场景的回放数据。
2. `src/utils/showcase-lab.ts`
    - 新增帧索引归一化、回放快照合成、自动推进逻辑。
3. `src/components/home/sections/ShowcaseLab.astro`
    - 新增回放控制区。
    - 渲染时使用“场景基线 + 回放帧覆盖”的合成结果。
    - 通过定时器驱动自动播放。
4. `tests/unit/showcase-lab.test.ts`
    - 验证回放快照与推进逻辑。
5. `tests/e2e/showcase-lab.spec.ts`
    - 验证播放按钮、进度条、自动推进和手动暂停行为。

## 风险与对策

1. 风险：时间驱动测试容易不稳定。
   对策：把每场景帧数控制在较小范围，并将帧间隔收敛到固定值，E2E 只验证关键跳转。

2. 风险：回放覆盖逻辑和场景基线分叉。
   对策：通过 `getShowcaseReplaySnapshot()` 统一生成 UI 所需视图数据。

3. 风险：自动播放抢占用户交互。
   对策：任何手动操作都立即暂停自动播放。

## 验收标准

1. 首页出现新的 `Mission Replay` 控制区。
2. 手动切帧时，指标、车位和链路状态同步变化。
3. 自动播放能从当前场景推进到下一个场景。
4. 手动操作会暂停自动播放。
5. 单测、构建和目标 E2E 全部通过。
