## 代码片段库规范

- 片段用途清晰，包含输入/输出与适用场景
- 统一命名，按模块与用途分目录
- 每个片段附最小示例与注意事项

**片段头部建议（示例）**

```
用途：
输入：
输出：
依赖：
适用场景：
```

## 命名规范（统一规则）

- 文件名建议使用英文或拼音 + 数字，统一小写
- 空格用 `-`，分段用 `_`
- 日期格式：`YYYYMMDD`
- 版本格式：`v1.0` / `v1.1`，重大变更升级主版本

## 命名模板（按类型）

- 文档/会议记录：`YYYYMMDD_topic_owner.md`
- 方案/评审记录：`YYYYMMDD_topic_review_vX.md`
- 数据集：`project_sensor_date_scene_vX`
- 模型权重：`project_task_data_epochN_vX`
- CAD/STEP：`car_system_part_vX.step`
- 线束/原理图：`module_schematic_vX.pdf`
- BOM：`BOM_car_YYYYMMDD_vX.xlsx`
- 代码片段：`module_purpose_snippet_vX.py`
- 脚本/工具：`module_purpose_YYYYMMDD.py`

## 索引字段规范

- 名称、路径、负责人、版本、更新日期、标签、状态（使用中/弃用/归档）

## 归档与弃用

- 过期资料移入 `/归档/` 并在索引中标记 `deprecated`
- 关键版本保留 README 说明变更点
