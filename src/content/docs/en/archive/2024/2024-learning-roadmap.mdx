## Before You Start

This section introduces the software and development platforms most new members will touch first.

<CardGrid>
    <Card title="ROS" icon="rocket">
        Robot Operating System is the main middleware used by the team. It is the toolbox that connects sensing, planning, control, and debugging workflows.
    </Card>
    <Card title="Linux" icon="linux">
        ROS development is most natural on Linux. Ubuntu remains the standard starting point for new members.
    </Card>
    <Card title="VS Code" icon="seti:code">
        The team commonly uses VS Code, though any editor or IDE you can use efficiently is acceptable.
    </Card>
    <Card title="C++ / Python" icon="seti:python">
        These are the two main languages used in our ROS-based work. Learn enough of both to read and experiment confidently.
    </Card>
</CardGrid>

### Learning Expectations

During onboarding, the team expects you to build three habits:

- search first and stay curious when you hit something unfamiliar
- think independently instead of copying tutorials mechanically
- ask good questions after you have made a serious attempt to investigate the answer yourself

<Aside type="tip" title="Training Philosophy">
Training is not about ticking boxes. What matters is that you actually learn how to think, search, test, and iterate as an engineer.
</Aside>

## 1. Base Environment Setup

<Steps>

1. **Choose an Ubuntu setup path**

    ROS works best on Ubuntu, so start by choosing how you want to run a Linux environment.

2. **Install Ubuntu**

    Pick the setup method that matches your machine and experience level.

3. **Learn the command line**

    Basic terminal literacy is mandatory for ROS, Git, and build workflows.

4. **Install and configure VS Code**

    Set up an editor you can actually use for debugging and development.

</Steps>

![Ubuntu logo](@assets/docs/archive/2024/2024-learning-roadmap/ubuntu-logo.webp)

### Ubuntu Setup Options

<Tabs>
    <TabItem label="VMware" icon="seti:config">
        **Best for beginners.** Running Ubuntu in VMware is simple, isolated, and forgiving when you are still learning. A practical starting point is 4 GB or more of RAM and 40 GB or more of disk space.
    </TabItem>

    <TabItem label="WSL2" icon="seti:windows">
        **Good for modern Windows users.** WSL2 starts quickly and integrates well with Windows, though some Linux GUI tools can still be awkward.
    </TabItem>

    <TabItem label="Dual Boot" icon="linux">
        **Best performance.** Installing Ubuntu directly gives you the cleanest Linux experience, but it is riskier and better suited to students who are already comfortable managing disks and bootloaders.
    </TabItem>

    <TabItem label="Docker" icon="seti:docker">
        **Fast environment replication.** Good if you already understand containers and want reproducible setups.
    </TabItem>

</Tabs>

### Linux Command Line Basics

Assume Ubuntu is installed and running. The next step is learning how to work in a terminal.

<Card title="Recommended Tutorial" icon="open-book">
    [Linux Hitchhiker's Guide](https://libhitchhiker.eu.org/) is a strong beginner-friendly reference.
</Card>

#### Where to enter commands

On an Ubuntu desktop:

- press `Ctrl` + `Alt` + `T` to open a terminal
- or open the application menu and search for `terminal`

<Aside type="tip">If you use WSL, the terminal window you open is already your Ubuntu shell.</Aside>
