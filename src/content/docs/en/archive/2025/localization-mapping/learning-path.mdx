## Phase 1: Coordinate transforms and spatial synchronization

### Overview

- understand the basic concept and common applications of **SLAM (Simultaneous Localization and Mapping)**
- understand coordinate frames, pose representation, and kinematic basics
- understand the difference between global and local localization and when each is used
- configure a ROS-based development environment and common tools
- achieve **spatial synchronization** across all sensors so their data aligns in a shared global frame

<Aside type="note">
In our team's pipeline, this is effectively part of the SLAM backend. That is why the emphasis is placed here.
</Aside>

### Coordinate-transform overview

#### 1. WGS84 to ECEF

Geodetic coordinates such as WGS84 are Earth-centered. ECEF uses the Earth's center as the origin, with the X axis pointing to the Greenwich meridian on the equator and the Z axis pointing to the North Pole.

Conversion formula:

```
X = (N(φ) + h) * cos(φ) * cos(λ)
Y = (N(φ) + h) * cos(φ) * sin(λ)
Z = (N(φ) * (1 - e²) + h) * sin(φ)
```

Where:
- `N(φ)` is the prime vertical radius of curvature
- `h` is altitude
- `φ` is latitude
- `λ` is longitude
- `e` is the first eccentricity of the Earth ellipsoid

#### 2. ECEF to ENU

ENU is a local tangent-plane coordinate system. The origin is typically chosen at a reference point such as the start point, with X east, Y north, and Z upward.

Conversion formula:

```
X_enu = X - X_origin
Y_enu = Y - Y_origin
Z_enu = Z - Z_origin
```
