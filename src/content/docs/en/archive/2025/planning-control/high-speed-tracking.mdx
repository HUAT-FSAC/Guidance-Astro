  </Card>
  <Card title="Curve Fitting" icon="pencil">

  </Card>

## Practical guidance

The planning stack runs inside ROS, and the quality of the generated path can be inspected directly by running the package.

**Suggestions**
- make sure the code runs end to end before trying to optimize it
- change the YAML parameters and compare the effect
- use MATLAB plotting if you need a clearer view of the path points
- once you understand the code, the origin of the planned points becomes much clearer

## Notes

<Aside type="tip">
It is normal for detected cone positions to be imperfect.
- severely incorrect cone positions are often filtered earlier in the Delaunay stage
- losing a very small number of cones does not necessarily break the plan
- as long as the road-cone structure is still recognizable, the system can often keep working
</Aside>
