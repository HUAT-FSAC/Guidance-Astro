## Why choose clangd?

The default VS Code `C/C++` extension often feels weak for ROS C++ development.

<CardGrid stagger>
    <Card title="Weak include indexing" icon="warning">
        Even if your `CMakeLists.txt` already declares the right libraries, you may still need to add paths manually in configuration.
    </Card>
    <Card title="Poor static analysis" icon="warning">
        Without accurate indexing, diagnostics and code understanding become unreliable.
    </Card>
    <Card title="Limited quick fixes" icon="warning">
        Error hints are often shallow and not very helpful in larger ROS workspaces.
    </Card>
</CardGrid>

<Aside type="tip">
    Using **clang + clangd** with VS Code gives a much better editing experience for ROS C++ projects.
</Aside>

## Install the Clang / clangd backend

The example below uses Ubuntu 18.04.

<Steps>

1. **Install clang**

    ```bash
    sudo apt install clang
    # depending on the Ubuntu repository state, this may be clang-10 instead
    ```

2. **Install clangd**

    ```bash
    sudo apt install clangd-10
    ```

    Create a symlink so the VS Code extension can find it:

    ```bash
    sudo ln -s /usr/bin/clangd-10 /usr/bin/clangd
    ```

3. **Install clang-format (optional)**

    ```bash
    sudo apt install clang-format
    ```

</Steps>

## Configure clangd

<Steps>

1. **Set the compile commands location**

    In the VS Code clangd extension settings, add this under `Clangd: Arguments`:

    ```
    --compile-commands-dir=${workspaceFolder}/build
    ```

    ![clangd-setting](@assets/docs/archive/ros-vsc-setup/clangd-setting.png)

2. **Export compile commands when building**

    <Tabs>
        <TabItem label="catkin build">
            ```bash
            catkin config --cmake-args -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
            catkin build
            ```
        </TabItem>
        <TabItem label="catkin_make">
            ```bash
            catkin_make -DCMAKE_EXPORT_COMPILE_COMMANDS=1
            ```
        </TabItem>
    </Tabs>

3. **Verify the result**

    Check whether `build/compile_commands.json` exists.

    <Aside type="tip">
        If the file is generated separately inside package build directories, you can merge them with the approach discussed in this [catkin_tools issue](https://github.com/catkin/catkin_tools/issues/551#issuecomment-553521463).
    </Aside>

</Steps>
