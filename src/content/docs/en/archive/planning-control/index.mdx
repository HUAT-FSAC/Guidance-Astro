## Overview

<video controls style="width: 100%; border-radius: 12px; margin: 1rem 0;">
    <source src={showcaseVideo} type="video/webm" />
</video>

Using perception outputs as input, the planning and control group turns track observations into drivable trajectories and executable control commands.

<Aside type="tip" title="Core Role">
    This group bridges perception and actuation by converting cone detections into paths, speed targets, and steering commands.
</Aside>

## Technical Architecture

```
Perception -> Boundary Detection -> Path Planning -> Trajectory Optimization -> Tracking Control -> Actuation
                 |                    |                    |
              cone clustering      Delaunay             Pure Pursuit
              boundary fitting     spline fitting       MPC
```

<Aside type="note">The planning and control stack is modular, so each stage can be debugged and optimized independently.</Aside>
