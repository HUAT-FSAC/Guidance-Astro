## Resource Scope

- Downloads: installers, drivers, toolchains, images
- Datasets: collection, labeling, and evaluation assets
- CAD and mechanical drawings: STEP, drawings, and BOMs
- Electrical documents and harness assets: schematics, harness diagrams, BOMs
- Code snippets: reusable scripts and functions
- Version index: asset versions and compatibility notes

## Unified Media Location

- Documentation images and videos live under `src/assets/docs/`
- Create subdirectories by topic or season and follow the naming rules below

## Suggested Directory Structure

## Code Snippet Rules

- State the purpose clearly
- Document inputs, outputs, and the intended use case
- Organize snippets by module and purpose
- Add one minimal example and one caution note per snippet

**Recommended snippet header**

```text
Purpose:
Input:
Output:
Dependencies:
Use case:
```

## Naming Rules

- Prefer lowercase English or pinyin with numbers
- Use `-` for spaces and `_` for segment grouping
- Date format: `YYYYMMDD`
- Version format: `v1.0`, `v1.1`, and bump the major version for large changes

## Naming Templates

- Documents / meeting notes: `YYYYMMDD_topic_owner.md`
- Reviews / proposals: `YYYYMMDD_topic_review_vX.md`
- Datasets: `project_sensor_date_scene_vX`
- Model weights: `project_task_data_epochN_vX`
- CAD / STEP: `car_system_part_vX.step`
- Schematics / harness: `module_schematic_vX.pdf`
- BOM: `BOM_car_YYYYMMDD_vX.xlsx`
- Code snippets: `module_purpose_snippet_vX.py`
- Tools / scripts: `module_purpose_YYYYMMDD.py`

## Index Fields

- Name, path, owner, version, updated date, tags, and status (`active`, `deprecated`, `archived`)

## Archive and Deprecation

- Move outdated material into `/archive/` and mark it as `deprecated` in the index
- Keep a short README with the reason and the important version differences
