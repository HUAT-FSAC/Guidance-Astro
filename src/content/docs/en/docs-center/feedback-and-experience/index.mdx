## Search and Tagging System

**Recommended tag dimensions**

- Group: algorithms / electrical / mechanical / operations
- Scenario: competition / training / on-vehicle / simulation
- Technology: perception / localization / planning / harness / suspension
- Asset type: dataset / CAD / schematic / script
- Difficulty: beginner / intermediate / advanced

**Tag rules**

- Keep tags lowercase and consistent
- Use 3 to 6 tags per page
- Let high-frequency keywords appear naturally in the title and summary

**Search-oriented writing**

- Put the core noun phrase early in the title
- Include synonyms and abbreviations in the first paragraph
- Break large topics with explicit section headers

## Reading Progress and Learning Paths

**Reading progress guidance**

- Split long documents into 3 to 5 sections
- Add a completion checkpoint per section
- Show reading time and the learning target on overview pages

**Learning path template**

- Beginner path: concepts -> toolchain -> small task
- Intermediate path: module practice -> optimization -> evaluation rules
- Project path: task breakdown -> solution review -> delivery -> retrospective

## Feedback Intake and Correction Flow

**Recommended intake points**

- Add a feedback button at the bottom of each doc page
- Standardize on one form or one issue template
- Let the operations group collect and triage requests centrally

**Correction flow**

1. Submit feedback with the page link
2. The owner confirms and assigns the fix
3. Mark the update in the change record after revision

## Metrics and Analysis

- Views and dwell time to estimate content value
- Search hit rate and zero-result rate to measure discoverability
- Navigation click-through and bounce rate to evaluate information structure
- Feedback adoption rate and revision cycle time to evaluate response speed
