## Standard Delivery Flow

| Stage | Purpose | Entry condition | Exit condition |
| --- | --- | --- | --- |
| Request / problem intake | Define the goal and scope | Source is clear | A task checklist exists |
| Solution review | Reduce implementation risk | First draft is ready | Review outcome is explicit |
| Execution & verification | Produce usable output | Task breakdown is ready | Self-check and validation pass |
| Delivery & acceptance | Confirm quality and archive results | Acceptance package is submitted | Acceptance passes and is archived |
| Retrospective & archive | Capture learning | Delivery is complete | Actions are recorded and closed |

## Lightweight Flow for Small Changes

1. Submit the task checklist
2. Request owner review
3. Update the content and archive it
