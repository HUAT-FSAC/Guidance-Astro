## 30-Minute Quick Start

- [ ] Open `/docs-center/` and `/2025/` to understand the current season structure
- [ ] Complete account setup for email, GitHub, and shared storage
- [ ] Join the team communication channels and set a consistent display name
- [ ] Install the baseline toolchain (see `/archive/general/`)
- [ ] Confirm your group direction and direct owner

## Finish on Day 1

- [ ] Read the team introduction, competition context, and recent outcomes
- [ ] Complete one minimal task or read one core doc and write notes
- [ ] Learn the resource-hub structure and naming rules
- [ ] Register this week’s target in the task checklist

## Week 1

- [ ] Finish environment setup and save verification notes
- [ ] Join one review session or document review
- [ ] Produce one archive-ready deliverable such as a document, script, drawing, or meeting note
- [ ] Define your learning path and stage goals

## Month 1

- [ ] Deliver one small module or subsystem task
- [ ] Submit one acceptance package and archive it
- [ ] Join one retrospective and record action items
- [ ] Be able to help another newcomer complete the 30-minute quick start

## Accounts and Access

| Item | Purpose | How to get it |
| --- | --- | --- |
| School email | Identity and account recovery | Issued by the school |
| GitHub organization | Code collaboration and PRs | Added by the owner |
| Shared storage | Documents and large files | Granted by the documentation admin |
| Docs system | Content editing and publishing | Assigned by the owner |

## Role Entry Points and First Task Suggestions

### Algorithms

- Entry points: `/2025/感知/`, `/2025/定位建图/`, `/2025/规控/`, `/2025/仿真测试/`
- First-week task: reproduce one basic workflow or read one module design doc
- Output: run log, issue list, and short summary

### Electrical

- Entry point: `/2025/电气/`
- First-week task: understand the electrical architecture and harness or schematic layout
- Output: architecture notes and open questions

### Mechanical

- Entry point: `/2025/机械/`
- First-week task: read the structure overview and open one CAD assembly
- Output: structure notes and risk list

### Operations

- Entry point: `/2025/项管/`
- First-week task: understand outward-facing materials and the operations workflow
- Output: material inventory and optimization suggestions

## Tools and Rules

- Naming format: `YYYYMMDD_topic_owner_vX`
- Archive location: `/en/docs-center/resource-hub/`
- Submission workflow: `/en/docs-center/processes-and-templates/`

## New Member Acceptance Checklist

- [ ] Can find the right group entry and responsible owner
- [ ] Can complete one minimal task and archive it independently
- [ ] Can submit a task checklist and acceptance note
- [ ] Can explain the resource-hub structure and naming rules

## Common Questions

- Don’t know where to start: finish the 30-minute quick start and confirm direction with your owner
- Environment setup failed: post logs and screenshots in the relevant channel
- Can’t find a file: check the resource hub first, then contact the documentation admin
