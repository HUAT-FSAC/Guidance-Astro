## 搜索与标签体系

**标签维度建议**

- 部门：算法 / 电气 / 机械 / 项管
- 场景：竞赛 / 训练 / 实车 / 仿真
- 技术：感知 / 定位 / 规控 / 线束 / 悬架
- 资产：数据集 / CAD / 原理图 / 脚本
- 难度：入门 / 进阶 / 高阶

**标签规则**

- 统一小写或拼音，避免混用中英文
- 标签数量建议 3-6 个
- 标题与摘要中自然出现高频关键词

**搜索优化写作**

- 标题前 10 个字放关键名词
- 首段包含同义词与缩写
- 使用清晰的小标题拆分主题

## 阅读进度与学习路线

**阅读进度建议**

- 长文拆分为 3-5 个小节
- 每节设置“完成检查点”
- 概览页给出阅读时长与目标

**学习路线模板**

- 入门路线：基础概念 → 工具链 → 小任务
- 进阶路线：模块实战 → 性能优化 → 评测规范
- 项目路线：需求拆解 → 方案评审 → 实现交付 → 复盘

## 反馈入口与纠错流程

**入口建议**

- 文档页底部放反馈按钮
- 统一反馈表单或 Issue 模板
- 运营群收集后集中录入

**纠错流程**

1. 提交反馈并定位页面链接
2. 负责人确认并分派
3. 修订完成后在更新记录中标注

## 数据分析与指标

- 阅读量与停留时长（衡量内容价值）
- 搜索命中率与零结果率（衡量检索体验）
- 目录点击率与跳出率（衡量结构合理性）
- 反馈采纳率与修订周期（衡量响应效率）
