# 内容贡献指南

欢迎为 HUAT FSAC 文档中心贡献内容！本文档将指导你如何有效地参与到项目中来，无论是提交新内容、修改现有文档还是修复错误。

## 贡献方式

### 1. GitHub Pull Request

这是最推荐的贡献方式，适合对文档进行较大的修改或添加新内容。

#### 步骤

1. **Fork 仓库**
   - 访问 [HUAT-FSAC 仓库](https://github.com/HUAT-FSAC/huat-fsac-docs)
   - 点击右上角的 "Fork" 按钮，创建自己的仓库副本

2. **克隆仓库**
   ```bash
   git clone https://github.com/你的用户名/huat-fsac-docs.git
   cd huat-fsac-docs
   ```

3. **创建分支**
   ```bash
   git checkout -b feature/你的功能名称
   ```

4. **安装依赖**
   ```bash
   pnpm install
   ```

5. **进行修改**
   - 添加新内容或修改现有文档
   - 确保代码风格一致
   - 测试你的更改

6. **提交更改**
   ```bash
   git add .
   git commit -m "描述你的更改"
   ```

7. **推送到 GitHub**
   ```bash
   git push origin feature/你的功能名称
   ```

8. **创建 Pull Request**
   - 回到 GitHub 上你的 fork 仓库
   - 点击 "Compare & pull request" 按钮
   - 填写 PR 描述，说明你的更改内容和原因
   - 点击 "Create pull request"

### 2. GitHub Issues

如果你发现了问题但不确定如何修复，或者有新功能建议，可以创建一个 Issue。

#### 步骤

1. 访问 [Issues 页面](https://github.com/HUAT-FSAC/huat-fsac-docs/issues)
2. 点击 "New issue" 按钮
3. 选择合适的模板（bug 报告或功能请求）
4. 填写详细信息
5. 点击 "Submit new issue"

### 3. 直接编辑

对于小的修改，你可以直接在 GitHub 上编辑文件。

#### 步骤

1. 找到你要修改的文件
2. 点击文件右上角的铅笔图标
3. 进行修改
4. 填写提交信息
5. 点击 "Propose changes"
6. 按照提示创建 Pull Request

## 内容规范

### 1. 文档结构

- **Markdown 格式**：所有文档使用 Markdown 格式
- **文件命名**：使用小写字母，单词之间用连字符 `-` 分隔
- **目录结构**：按照功能模块和年份组织内容

### 2. 内容要求

- **准确性**：确保内容准确无误
- **完整性**：提供完整的信息，包括必要的背景和步骤
- **清晰度**：使用简洁明了的语言
- **一致性**：保持与现有文档风格一致
- **引用**：如果引用了外部资源，请提供链接

### 3. 代码规范

- **代码块**：使用代码块展示代码示例
- **语法高亮**：为代码块添加语言标识
- **注释**：为复杂代码添加注释
- **格式**：保持代码格式整洁

## 审核流程

1. **提交 PR**：作者提交 Pull Request
2. **自动化检查**：系统自动运行代码质量和构建检查
3. **人工审核**：维护者审核更改内容
4. **反馈**：维护者提供反馈，作者进行修改
5. **合并**：审核通过后，PR 被合并到主分支
6. **部署**：更改会自动部署到网站

## 贡献者指南

### 1. 行为准则

- 尊重其他贡献者
- 接受建设性批评
- 关注问题的解决方案
- 保持专业和友好的态度

### 2. 最佳实践

- **小而专注**：每个 PR 只解决一个问题
- **描述清晰**：提供详细的 PR 描述
- **测试充分**：确保你的更改不会破坏现有功能
- **遵循规范**：按照项目的代码和文档规范进行贡献

### 3. 常见问题

#### Q: 如何处理合并冲突？
A: 在提交 PR 前，先从主分支拉取最新代码并解决冲突。

#### Q: 我的 PR 被拒绝了怎么办？
A: 不要灰心！维护者会提供具体的反馈，根据反馈修改后再次提交。

#### Q: 我可以贡献哪些类型的内容？
A: 你可以贡献技术文档、教程、经验分享、项目进度更新等任何与 HUAT FSAC 相关的内容。
